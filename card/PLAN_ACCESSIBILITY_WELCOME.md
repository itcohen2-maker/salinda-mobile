# תוכנית הנגשה וברוכים הבאים — מסך השחקן ומשתמש ראשון

## מטרה
להנגיש את המשחק למשתמש שנכנס **פעם ראשונה** ולא יודע מה לעשות: להציג ברוכים הבאים והסבר קצר במסך השחקן (מעבר תור / תחילת משחק), ולחזק עידוד חיובי (חצי הכוס המלאה).

---

## שלב 1: ברוכים הבאים במסך השחקן (TurnTransition)

### בעיה
- אין שום התראה או הסבר כשמשתמש נכנס לראשונה למסך "עכשיו התור שלך" (TurnTransition).
- משתמש חדש לא יודע שצריך להזין שם, לראות את הקלפים, או ללחוץ "אני מוכן".

### ביצוע
- [x] **מפתח AsyncStorage:** `salinda_welcome_player_screen_seen` (או כחלק מ־onb_welcome_screen).
- [x] **תנאי:** רק אם המפתח לא נשמר (פעם ראשונה שנכנסים ל־TurnTransition במשחק הזה או אי־פעם).
- [x] **תוכן:** קופסה בסגנון alertBubble (חומה מעוגלת, כותרת צהובה, גוף אפור):
  - כותרת: **"👋 ברוכים/ות למשחק!"**
  - גוף: **"זה התור שלך. מלא/י את שמך למעלה (פעם אחת), הכר את הקלפים, ולחץ/י 'אני מוכן' כשאת/ה מוכן/ה להתחיל."**
- [x] **סגירה:** כפתור "הבנתי" או סגירה אוטומטית אחרי 6–8 שניות; בשמירת המפתח כדי לא להציג שוב. *(מומש: כפתור "הבנתי" + שמירת מפתח; בלי סגירה אוטומטית לבועה הזו.)*
- [x] **מיקום:** מעל או מתחת לשדה השם, לא חוסם את "אני מוכן".

## שלב 2: ברוכים במסך המשחק (pre-roll) — חיזוק

### מצב קיים
- כבר קיים `onb_game_start`: "ברוכים הבאים למשחק! המטרה: היפטרו מכל הקלפים..."
- מוצג רק כשנכנסים ל־pre-roll **בפעם הראשונה** (ותלוי בהדרכה מופעלת).

### ביצוע
- [x] לוודא שההתראה **באמת מופיעה** בכניסה ראשונה ל־pre-roll (לא נחסמת על ידי tutorial/identicalAlert). *(ברוכים מ־GameProvider + `requireAck`; הודעות onb אינן חסומות ע"י tutVisible.)*
- [x] אם רוצים הדגשה נוספת: להזכיר במשפט אחד — "אם יש לך קלף זהה לערימה, אפשר להניח אותו ממש בהתחלה ולוותר על הטלת קוביות." *(בגוף ההודעה ב־GameProvider וב־`showOnb(onb_game_start)`.)*

## שלב 3: קלף זהה — חצי הכוס המלאה

### רקע
- ההתראה "יש לך קלף זהה ?" ליד הערימה **מאשרת** שאפשר להניח את הקלף, להיפטר ממנו ולהעביר את התור.
- ממש **בהתחלה**, לפני הטלת קוביות, אפשר להניח קלף זהה — וזה **עידוד**: יש לנו פחות קלף.

### ביצוע
- [x] **אחרי לחיצה על "הבנתי!" ב־identicalAlert:** להוסיף התראה קצרה (או לשלב בהודעת המעבר):
  - **"🎉 יש לנו פחות קלף אחד!"** (או "פחות קלף — חצי הכוס המלאה!").
- [x] **מימוש:** ב־`DISMISS_IDENTICAL_ALERT` או ב־endTurnLogic — להוסיף ל־notifications הודעה עם `style: 'success'` ו־`autoDismissMs: 2500`.
- [x] כך המשתמש רואה בבירור את "חצי הכוס המלאה" — פחות קלף ביד.

## שלב 4: קלף זהה ותוצאות אפשריות

### תצוגה
- ההתראה "קלף זהה" ליד הערימה **עשויה לכסות או להסתיר** את כפתור "תוצאות אפשריות" כששניהם רלוונטיים.
- זה **מכוון**: בהתחלה לפני הטלת קוביות, הדגש הוא על האפשרות להניח קלף זהה; "תוצאות אפשריות" רלוונטי יותר אחרי הטלה.

### ביצוע
- [x] לא לחייב שינוי — לאשר שהתנהגות נוכחית (קלף זהה בולט בהתחלה) מתאימה.
- [x] אופציונלי: אם רוצים — כשמסתירים את ההתראה "קלף זהה" (חץ/בועה), לוודא ש"תוצאות אפשריות" נגיש מיד (ללא עיכוב). *(ללא שינוי נוסף.)*

## שלב 6: סיכום הנגשה

| מקום | פעולה |
|------|--------|
| **TurnTransition** | ברוכים + הסבר "מה עושים עכשיו" (פעם ראשונה) |
| **Pre-roll (GameScreen)** | לוודא onb_game_start מופיע; אופציונלי: להזכיר קלף זהה בהתחלה |
| **אחרי קלף זהה** | הוספת "🎉 יש לנו פחות קלף אחד!" לעידוד |

| **אחרי הטלה → building** | התראת מידע חד־פעמית + צליל בועה |
