# חדר הזהב — סבב תרגול קוביות-משוואה

תאריך: 2026-05-29
סטטוס: עיצוב מאושר (חלקים 1+2), ממתין לאישור ספק כתוב

## מטרה

הוספת Flow תרגול חדש בחדר הזהב שבו המשתמש לומד **לתפעל משוואה** ברמת
הבסיס (חיבור וחיסור בלבד). אין שום Hardcoding של ערכי קוביות — בכל סבב
נוצרות 3 קוביות אקראיות חדשות דרך `generateDiceSet()`, עם "שומר סף"
(Validator) שמוודא שאפשר לבנות מהן לפחות משוואה חוקית אחת, כך שהמשתמש
לעולם לא נתקע מול "קיר" בלתי אפשרי.

ה-Flow: בחירת קלף מהערימה ← אנימציית קוביות נופלות (עם הסלוגן "לחשוב
מחוץ למשוואה..." כאקט מיתוג בזמן ההמתנה) ← פתיחת Slots לבניית המשוואה.

## החלטות מפתח (מאושרות)

- **פעולות ברמת הבסיס:** `+` ו-`−` בלבד. "זאת הרמה הכי נמוכה שאפשר;
  מי שיודע כפל יודע חילוק" — כפל/חילוק שמורים לרמה גבוהה יותר בעתיד.
  הפונקציה נבנית מודעת-לפעולות מהיום הראשון כדי לתמוך ברמות הבאות.
- **מיקום (ברירת מחדל):** משימה/אריח חדש ברשת המשימות של חדר הזהב
  ("תרגול משוואות 🎲"), עצמאי ולא נוגע בלוגיקת המשחק החי.
- **תפקיד הקלף (ברירת מחדל):** טריגר להתחלת הסבב בלבד — אין `target`;
  המשתמש בונה כל משוואה חוקית לתרגול. (הפונקציה תומכת ב-`target`
  אופציונלי לשימוש עתידי.)
- **שימוש חוזר ברכיבים קיימים:** הרכיב החדש הוא תזמורת בלבד. אפס שכפול
  של לוגיקת קוביות או משוואה.

## חלק 1 — `generateDiceSet()` (לוגיקה טהורה, TDD)

קובץ חדש: `src/goldroom/generateDiceSet.ts`. ללא React. נכתב לפי TDD.

### חוזה (API)

```ts
type Operation = '+' | '-' | 'x' | '÷';

interface GenerateDiceOptions {
  operators?: Operation[];   // ברירת מחדל: ['+', '-']
  target?: number;           // אופציונלי: דרוש שתוצאה אחת תשווה לערך הזה
  allowNegative?: boolean;   // ברירת מחדל: false
  maxAttempts?: number;      // ברירת מחדל: 50
  rng?: () => number;        // הזרקה לבדיקות; ברירת מחדל Math.random
}

// מחזיר שלשה אקראית מאומתת [1..6, 1..6, 1..6]
function generateDiceSet(opts?: GenerateDiceOptions): [number, number, number];

// עטיפה דקה עם דיפולטי רמת הבסיס (+ ו- בלבד)
function generateTutorialSeed(): [number, number, number];
```

### לוגיקת ה-Validator (שומר הסף)

`hasValidEquation(dice, operators, opts)` — פונקציה פנימית שמשתמשת מחדש
ב-`applyOperation` הקיים מ-`src/utils/arithmetic`:

- עוברת על כל התמורות (permutations) של 3 הקוביות.
- לכל זוג פעולות (op1, op2) מתוך `operators`, ולשתי האסוציאטיביות
  `(a op1 b) op2 c` ו-`a op1 (b op2 c)`.
- מחזירה `true` אם קיימת תוצאה אחת לפחות שהיא מספר שלם, ושהיא `>= 0`
  (אלא אם `allowNegative`), ושהיא שווה ל-`target` אם `target` סופק.
- כדי לשקף את ה-Flow שבו המשתמש כן רשאי לבנות משוואת 2-קוביות, ה-validator
  בודק גם צירופי 2-קוביות (a op b) — כך "חוקי" כאן תואם למה שה-Slots
  יקבלו בפועל.

### זרימת `generateDiceSet`

1. לולאה עד `maxAttempts`: הטל 3 קוביות (1-6) דרך `rng`; אם
   `hasValidEquation` → החזר אותן.
2. **Fallback בלי Hardcoding:** אם המכסה נגמרה, מנה את כל 216 הצירופים,
   סנן לאלה ש-`hasValidEquation` מחזיר עליהם `true`, ובחר אחד דרך `rng`.
   זה מבטיח החזרה של מערך חוקי תמיד — לעולם לא "קיר" — בלי ערכים קבועים.
   (אם, תיאורטית, אין אף צירוף חוקי תחת ה-operators/target הנתונים,
   נזרקת שגיאה ברורה — מצב שלא קורה עם הדיפולטים `+`/`−`.)

### הערת כנות על הדיפולטים

עם `+` ו-`−` בלבד וללא `target`, כמעט כל הטלה ממילא חוקית (חיבור תמיד
נותן שלם ≥0). ה-Validator הוא בעיקר רשת ביטחון נכונה לרמות עתידיות
(חילוק / יעד נדרש), ולכן נבנה מודע-לפעולות כבר עכשיו.

## חלק 2 — ה-Flow הויזואלי

קובץ חדש: `src/goldroom/DiceEquationRound.tsx`. מכונת-מצבים של סבב אחד.

```
pickCard  →  rolling  →  build
```

### מצב `pickCard`
ערימת קלפים מוצגת; המשתמש בוחר קלף → טריגר לתחילת הסבב.

### מצב `rolling`
- קריאה ל-`generateDiceSet({ operators: ['+','-'] })` → 3 ערכים מאומתים.
- רינדור `AnimatedDice` הקיים (`AnimatedDice.tsx`) עם:
  - `fixedFinalValues={validatedDice}` — הקוביות נוחתות בדיוק על הערכים
    המאומתים (התכונה כבר קיימת ברכיב).
  - `autoRollOnMount` + `hideRollButton` — הגלגול מתחיל לבד, בלי כפתור.
- בתחתית המסך בזמן הגלגול: הסלוגן **"לחשוב מחוץ למשוואה..."** עם
  אנימציית fade/pulse עדינה. מופיע בתחילת הגלגול, נעלם בעצירה.
- המעבר ל-`build` מופעל מ-`onRollComplete` של `AnimatedDice` (שכבר
  מחזיר את הערכים הסופיים).

### מצב `build`
- רינדור `EquationSlots` הקיים
  (`src/components/onboarding/EquationSlots.tsx`, אותו רכיב שב-
  `TutorialGameScreen`) עם 3 הקוביות שנחתו.
- המשתמש ממלא את ה-Slots ובונה משוואה — תרגול תפעול משוואה.

### עיקרון
הרכיב החדש מחבר בלבד בין `generateDiceSet` ↔ `AnimatedDice` ↔
`EquationSlots`. אין לוגיקת קוביות/משוואה משוכפלת.

### שילוב בחדר הזהב
אריח חדש ב-`TASKS` בתוך `src/goldroom/GoldRoomScreen.tsx`
(`{ id: 'equation-practice', badge: '🎲', title: 'תרגול משוואות', ... }`).
בחירתו פותחת את `DiceEquationRound` במקום זרימת ה-Spotlight הרגילה.
